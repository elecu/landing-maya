Features

māyā is a GUI workflow for collider studies, starting from detector-level Monte Carlo samples. It reads standard LHCO or CSV files (e.g. Delphes outputs), builds an analysis dataset, and takes you from raw events to ML driven sensitivity in a few steps.

  • Final-state definition & observables: choose object types and multiplicities (γ, e±, μ±, jets, MET), define pairs/triplets/quartets, and compute key variables such as invariant mass, transverse mass (with MET), ΔR separations and related kinematic quantities.
  • Event processing & plots: load multiple signal/background files, optionally filter events that fail your object requirements, export processed CSVs, and visualise distributions with overlaid histograms plus Gaussian, KDE or exponential fits for quick shape interpretation.
  • BDT training: train an XGBoost classifier for signal–background separation, with optional Optuna hyperparameter tuning. The tool outputs ROC and precision–recall curves, confusion matrices and feature-importance rankings. Missing values can be handled directly or via optional MiceForest imputation.
  • Sensitivity scan: apply manual cuts if desired, then scan classifier-score thresholds and compute expected significances S/√(S+B) for your chosen cross sections and integrated luminosity, automatically finding the optimal working point.

Requirements

  • Operating systems: 64-bit Ubuntu 22.04+ or Fedora 38+, Windows 10/11, and macOS 13+ on Intel or Apple Silicon.
  • Internet connection: an active internet connection is required while using māyā.
  • macOS Intel (x64): some systems may require installing Apple’s Command Line Tools (xcode-select --install) before the app can run normally. You may also need to allow the app under System Settings → Privacy & Security.
  • macOS Apple Silicon (M1/M2/M3): if your system requests it, install Rosetta 2 (softwareupdate --install-rosetta) to run x86-64 binaries. This is a one-time setup.
